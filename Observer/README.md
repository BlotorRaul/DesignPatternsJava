# stocks.Observer Design Pattern
when you need many other objects to receive an update when another object changes

### Pro
Avantajele design pattern-ului stocks.Observer 
includ reducerea cuplajului între obiecte
(Cuplajul între obiecte se referă la cât 
de strâns sunt legate sau dependente între
ele în codul tău de programare. Cu cât două 
obiecte sunt mai strâns legate, cu atât este 
mai greu să modifici unul fără a afecta celălalt.) ,
deoarece subiectul nu trebuie să știe detalii despre
observatori, și modularitatea, deoarece puteți 
adăuga sau elimina observatori fără a modifica 
subiectul.
